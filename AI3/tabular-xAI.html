    Introduction

    The previous section (and much of this module) focusses on image data, however, the concepts are transferable to tabular data. In this section we will explore how features can be removed to help explain their attribution, as well as how importance can be inferred by integrating the gradients of an input given a zeroed baseline.

    Setup

    Data

    The dataset used in this demonstration is the Wisconsin Breast Cancer Diagnostic Database, publicly available from the University of California Irvine (UCI) Machine Learning Repository. It consists of characteristics of cell nuclei taken from breast masses which were sampled using fine-needle aspiration (FNA), a common diagnostic procedure in oncology.

    The features are observable characteristics of the cells in the images - for example, radius, concavity, and texture. An example of one of the digitised images from an FNA sample is given below.

    Neural Network Model

    We will use a simple neural network with two hidden layer and Tanh activation functions per each layer.

    class simpleNN(nn.Module):
    def __init__(self, in_features):
        super().__init__()

        self.network = nn.Sequential(

            # Layer 1
            nn.Linear(in_features, 20),
            nn.Tanh(),

            nn.Linear(20, 12),
            nn.Tanh(),

            nn.Linear(12, 2) # binary classification
        )

    def forward(self, x):

        logits = self.network(x)

        return logits
    NOTE

    Remember from AI1 that no Softmax layer is applied in the forward method, as PyTorch's nn.CrossEntropyLoss executes it internally when calculating the loss. For inference, use argmax with the raw logits to find the predicted class.

    Feature Ablation

    The concept of ablating parts of the information in the test dataset can be carried over to tabular data as well. This time, we can use the FeatureAblation class to remove features rather than pixels.

    from captum.attr import FeatureAblation

    Like with the previous examples, we will only use one sample to obtain a score.

    # Get ablation class and set model to evaluation mode
ablation = FeatureAblation(model)
model.eval() # set model to evaluation mode

# Get a single sample and its predicted class
sample = test_input_tensor[2:3]
predicted_class = model(sample).argmax().item()

# Calculate importances
attr = ablation.attribute(
    sample,
    target=predicted_class,
)

plt.figure(figsize=(10, 6))
plt.barh(feature_names, attr[0].detach().cpu())
plt.xlabel('Attribution Score')
plt.tight_layout()
plt.show()

    According to this attribution score most features are either positively effecting the prediction or negatively, with only a few around zero (which means they have no impact on the prediction). However, looking at the scale these values are very small, indicating that they may also have a minimal impact on the prediction. Due to the variation of just one sample, it is often best to obtain a more stable attribution by using multiple samples.

    TIP

    As a default, the ablation class will set the ablated feature to 0. However, it can be modified to be more specific. For example, using the mean of the feature from the training data is a common approach.

    baseline_mean = torch.tensor(X_train.mean(axis=0)).unsqueeze(0)

attr = ablation.attribute(
    sample,
    baseline=baseline_mean,
    target=predicted_class,
)

    Additionally to getting the scores for multiple samples, we can separate out the attributions for the positive class (malignant) and the negative class (benign) to see which features are most important for each class.

    # Collect attributions
all_attributions = {0: [], 1: []}

for class_idx in range(n_classes):
    
    # Get samples from this class
    class_mask = y_test == class_idx
    class_samples = X_test[class_mask]
    
    # Iterate over multiple samples, collecting their scores
    for sample in class_samples:
        sample_tensor = torch.tensor(sample, dtype=torch.float32, device=device).unsqueeze(0)
        attr = ablation.attribute(sample_tensor, target=class_idx)
        all_attributions[class_idx].append(attr.squeeze().cpu().numpy())

# Average attributions per class
avg_attributions_malignant = np.mean(all_attributions[1], axis=0)
avg_attributions_benign = np.mean(all_attributions[0], axis=0)
# Find features that are important for one class but not the other
diff_importance = avg_attributions_malignant - avg_attributions_benign

plt.figure(figsize=(10, 6)) 
colors = ['red' if x < 0 else 'green' for x in diff_importance]
plt.barh(feature_names, diff_importance, color=colors)
plt.xlabel('Importance Difference (Malignant minus Benign)')
plt.axvline(x=0, color='black', linestyle='--') # plot dashed line at 0
plt.tight_layout()
plt.show()

    Finding the average importance of each feature gives a much richer view of what features are most important to the model. According to this representation, radius2 is most important for predicting if a cell is benign with smoothness3 and concavity3 for malignancy most of the features are positively correlated with the target variable, malignant.

    Integrated Gradients

    An alternative to feature ablation is to calculate how much each feature contributes through their integrated gradients. This method measures how much each feature contributes to the final prediction by computing gradients along a path from a baseline (e.g., zeros or average values) to the actual input.

    Rather than simply removing features, Integrated Gradients creates many intermediate points between the baseline and your input, calculates how sensitive the model is to each feature at each point, and then averages these sensitivities.

    This gradient-based approach is often more efficient than ablation for high-dimensional data.

    from captum.attr import IntegratedGradients

    Differently to the previous class, we can give IntegratedGradients the whole test dataset in one go, setting the target to either 0 or 1. Here we have set it as 1, meaning positive values will indicate the feature contributes to the prediction of the malignant class and negative values means it supressess predictions of the malignant class.

    It is an important point to distinguish, negative values are not necessarily predictive of benign (although they may well be). To find which ones are we would have to change the target to 0.

    # Move to CPU just in case for apple users
model.cpu()
test_input_cpu = test_input_tensor.cpu()
test_input_cpu.requires_grad_()

ig = IntegratedGradients(model)

attr, delta = ig.attribute(test_input_cpu,
                            target=1, # calculate the gradients for the positive class (malignant)
                            return_convergence_delta=True
                            )

attr = attr.detach().numpy()
attr_means = np.mean(attr, axis=0)

model.to(device);  # move back to original device
    IMPORTANT

    In the above code we have to move the model temporarily to the CPU to prevent conflicts with any Apple silicone devices. This is because Captum uses Float64 in it's calculations here, which Apple silicone is unable to do. If you are working with a GPU or CPU already this is un-necessary.

    Distribution of scores

    Means of scores can often be misleading, especially for feature attribution methods where zero indicates no contribution to the prediction either positively or negatively. Plotting a histogram of the attributions can give a better sense of the distribution.

    fig, ax = plt.subplots(1, 2, figsize=(10, 4))

ax[0].hist(all_best);
ax[0].set_title(f"Distribution of most positive correlated feature: \n {positive_feature}");

ax[1].hist(all_worst);
ax[1].set_title(f"Distribution of most negative correlated feature: \n {negative_feature}");

fig.tight_layout()

plt.show()

    Both distributions for the most positive and negative predictions are centered around 0 as most samples have a feature attribution score of 0. This indicates that in the majority of cases, most values for these features do not contribute much at all. This could be likely for all features.

    Scores per value

    This distribution is to be expected, it is not the average values that are typically indicative of how one class differs from another, but rather the outliers and unique measurements. We therefore want to build a profile of how the range of values in a feature changes its contribution to the prediction.

    One way to do this is to split the range of values for a feature into discrete bins, each summarised by the count and mean attribution score of the samples within. For example, with smoothness3 we could take all values between 0 and 0.1, count them, and calculate their mean attribution score, before moving on to 0.1 and 0.2, and so on. Binning the results this way smooths out noisy variation, hopefully surfacing any underlying trends.

    from scipy import stats
    REMEMBER

    As we will now be working with the actual values, we will need to revert them back to their true scales due to the use of StandardScaler.

    # Revert the scaling
X_test_unscaled = scaler.inverse_transform(X_test)
    # Get the indexes of the best features
best_positive_idx = np.argmax(attr_means)
best_negative_idx = np.argmin(attr_means)
    # Bin the positive feature 
# Rather than plotting every individual sample, we group the feature values into
# 6 equal-width bins and summarise each bin by its mean attribution score and sample count. 
bin_means, bin_edges, _ = stats.binned_statistic(
    X_test_unscaled[:, best_positive_idx],   # feature values along the x-axis
    attr[:, best_positive_idx],     # corresponding attribution scores
    statistic='mean',
    bins=6
)
bin_count, _, _ = stats.binned_statistic(
    X_test_unscaled[:, best_positive_idx],
    attr[:, best_positive_idx],
    statistic='count',
    bins=6
)

# Bin the negative feature 
bin_means_neg, bin_edges_neg, _ = stats.binned_statistic(
    X_test_unscaled[:, best_negative_idx],
    attr[:, best_negative_idx],
    statistic='mean',
    bins=6
)
bin_count_neg, _, _ = stats.binned_statistic(
    X_test_unscaled[:, best_negative_idx],
    attr[:, best_negative_idx],
    statistic='count',
    bins=6
)

# Compute bin centre positions for plotting 
# binned_statistic returns bin *edges*, so we derive the centre of each bin by
# stepping back half a bin-width from each right edge.
bin_width = bin_edges[1] - bin_edges[0]
bin_centers = bin_edges[1:] - bin_width / 2

bin_width_neg = bin_edges_neg[1] - bin_edges_neg[0]
bin_centers_neg = bin_edges_neg[1:] - bin_width_neg / 2
    # Plot 
fig, ax = plt.subplots(1, 2, figsize=(12, 5))

ax[0].scatter(bin_centers, bin_means, s=bin_count)
ax[0].set_ylabel("Average Attribution")
ax[0].set_xlabel(positive_feature)
ax[0].set_title(f"Binned attribution for positively correlated feature\n{positive_feature}")

ax[1].scatter(bin_centers_neg, bin_means_neg, s=bin_count_neg)
ax[1].set_ylabel("Average Attribution")
ax[1].set_xlabel(negative_feature)
ax[1].set_title(f"Binned attribution for negatively correlated feature\n{negative_feature}")

plt.tight_layout()
plt.show()

    Breaking the scores down per bins of values reveals information about how the data contributes to the prediction. For the positively correlated feature, smoothness3, the average attribution rises as smoothness3 increases, indicating that higher values contribute to higher predictions of malignancy. Values between 0.16 - 0.18 are heavily predictive of a malignant diagnosis.

    For radius2, the same pattern is seen, however we interpret it differently as the negative values indicate those that supress the malignant prediction. Here, cells with a radius of 0.2 (the majority) suppress the malignancy prediction and likely has a high chance of predicting benign.

    The further breaking down of each feature allows us to extract meaningful biological information that can be utilised by a professional to better guide their understanding of the output or their research.

    Other Methods

    Captum has several other methods for better understanding neural networks. Two that are prominently used are the Layer Attributions class and the Neuron Attributions class. These facilitate insight into how each layer in the neural network contributed to the final prediction and how each feature inputs into each neuron.

    Layer Attributions allow you to examine the importance of each layer of the network for a given prediction. This is particularly valuable for debugging deep networks and understanding what patterns the model has learned at different levels of abstraction.

    Neuron Attributions go deeper by showing how individual neurons respond to specific inputs, revealing which input features activate particular neurons and how strongly. This granular view helps identify whether neurons have learned meaningful patterns or spurious correlations, and can guide network architecture improvements.

    Combined, these two methods can help break down every component of the model to best understand how it is giving predictions. They enable you to trace the flow of information from raw inputs through hidden layers to final outputs, making it possible to identify exactly where and how the model processes different types of information.

    Summary

    Gaining meaningful insights from machine learning models can be arduous, relying on domain knowledge combined with a deep dive into how the scale of values of a feature contribute to a prediction. Even when a feature that is important to a prediction is identified, it must be remembered that it is still a correlation, not a causation. There have been many cases of models with accurate predictions that were fundamentally flawed due to biases in their training data. It is up to researchers to identify these biases and rectify them, not to confirm them as true.
